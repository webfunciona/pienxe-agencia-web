# PIENXE — OPERATIONS PROTOCOL (V2.4)

## 1. VISION & IDENTITY
Pienxe no es una fábrica de software; es un **estudio de ingeniería de sistemas y orquestación agéntica**. Operamos bajo el modelo de "Boutique Técnica" desde Hamburgo, priorizando la arquitectura sólida sobre la moda tecnológica.

- **Filosofía**: El código es un pasivo; el sistema operando es el activo.
- **Promesa**: Lo que entregamos debe estar operando el lunes siguiente, o no se entrega.
- **Estética**: High-end, brutalismo técnico y rendimiento extremo (LCP < 1.2s).

## 2. THE STACK (CORE)
- **Frontend**: Astro (Static First, Islands of Interactivity).
- **Styling**: Vanilla CSS / Tailwind (Solo si el cliente lo pide expresamente).
- **Backend/Logic**: n8n (Orquestación), Python (Agentes/RAG), Supabase (Persistence).
- **Deployment**: Dokploy / VPS dedicados (Soberanía de datos).

## 3. SDD (SPEC-DRIVEN DEVELOPMENT)
No escribimos código sin una especificación. El flujo de trabajo obligatorio para cualquier cambio significativo es:
1. **Proposal**: Definición del problema y alcance.
2. **Spec**: Requerimientos técnicos y funcionales claros.
3. **Design**: Arquitectura y decisiones técnicas (trade-offs).
4. **Tasks**: Breakdown atómico para ejecución.
5. **Apply**: Implementación batch.
6. **Verify**: Validación contra la Spec original.

## 4. AGENTIC PROTOCOL
Pienxe utiliza una red de agentes especializados (CTO, CMO, UXDesigner) orquestados por un CEO agente.
- **Human-in-the-loop**: La IA propone y ejecuta; el humano valida y dirige.
- **Persistence**: Todo aprendizaje, decisión técnica o bug corregido DEBE persistirse en la memoria colectiva (Engram).

## 5. POLICIES
- **Cero Placeholders**: No usamos "lorem ipsum" ni imágenes de stock genéricas. Generamos activos reales.
- **SEO por Defecto**: Cada página debe ser semántica y cumplir con los estándares de Lighthouse 100/100.
- **Voseo Rioplatense**: En comunicaciones internas y (cuando aplique) externas en español, mantenemos una identidad directa y apasionada.
